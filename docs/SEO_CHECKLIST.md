# قائمة تحقق SEO — لظهور الموقع الجديد قبل القديم

## ما تم تنفيذه تلقائياً في الكود

1. **Meta Tags**: وصف افتراضي عند الفراغ، كلمات مفتاحية، Open Graph مع og:site_name و og:image:alt
2. **Structured Data**: schema WebSite + Organization عند عدم وجود إعداد مخصص
3. **Sitemap**: تضمين جميع الصفحات (الرئيسية، من نحن، المنح، الأخبار، كنعاني، تمكين، مظلات، الخصوصية، الشروط) + نسخ عربي وإنجليزي
4. **robots.txt**: دعم محتوى مخصص من لوحة التحكم، وإضافة Sitemap تلقائياً
5. **Canonical + hreflang**: روابط بديلة للغتين

---

## خطوات يدوية ضرورية لتحسين الترتيب

### 1. إعداد Google Search Console (أهم خطوة)
- ادخل إلى [Google Search Console](https://search.google.com/search-console)
- أضف الموقع الجديد كخاصية
- تحقق من الملكية عبر ملف HTML أو meta tag (من لوحة التحكم → إعدادات SEO)
- أرسل خريطة الموقع: `https://your-domain.com/sitemap.xml`

### 2. إعداد Bing Webmaster Tools
- ادخل إلى [Bing Webmaster Tools](https://www.bing.com/webmasters)
- أضف الموقع وتحقق منه
- أرسل خريطة الموقع

### 3. طلب الفهرسة للصفحات المهمة
- من Google Search Console: استخدم "فحص الرابط" (URL Inspection)
- اطلب الفهرسة للصفحة الرئيسية وأهم الصفحات

### 4. التأكد من إعدادات لوحة التحكم
- **إعدادات SEO** → تأكد أن "السماح بالفهرسة" مفعّل
- املأ الوصف التعريفي (meta description) والكلمات المفتاحية
- أضف Google Site Verification و Bing Verification إن لم تكن مضافة

### 5. إعداد تحويل 301 من الموقع القديم (إن أمكن)
- على استضافة bzufa.ps القديم: أعد تحويلاً 301 من كل صفحة إلى النسخة المقابلة في الموقع الجديد
- هذا ينقل "قوة" الترتيب من القديم إلى الجديد

### 6. تحسين سرعة التحميل
- ضغط الصور قبل الرفع
- تفعيل التخزين المؤقت (cache) على السيرفر
- استخدم CDN إن أمكن
- **ما تم تنفيذه في الكود**:
  - تحميل Swiper (السلايدر) فقط في الصفحة الرئيسية
  - lazy loading للصور تحت الطيّ (المشاريع، الشراكات، الأخبار، المنتجات)
  - width/height على الصور لتقليل CLS
  - تحميل الخطوط والـ CSS غير الحرجة بتأجيل (media="print" onload)
  - preconnect و dns-prefetch للنطاقات الخارجية

## ملاحظات

- تحديث المحتوى بانتظام يساعد في التحسين
